○
¿Por qué es importante este paso?

Excel tiene dos formatos principales: .xlsx (sin macros) y .xlsm (con macros habilitadas). Si guardas en .xlsx, todo tu código VBA se perderá. El formato .xlsm es el contenedor que almacena tanto los datos como el código que escribiremos.

Insertar un Módulo estándar y escribir el encabezado
El módulo es donde vivirá el código que abre el formulario
VBA
○
¿Por qué necesitamos un Módulo?

En VBA hay dos lugares para escribir código: dentro de un UserForm (que usaremos para los botones del formulario) y dentro de un Módulo estándar (para procedimientos generales). El módulo es el lugar donde pondremos el procedimiento que abre el formulario cuando el usuario haga clic en un botón en la hoja de Excel.

Los comentarios en VBA comienzan con apóstrofo ('). Son texto que el programa ignora, pero que sirve para documentar qué hace cada parte del código.

📍
Dónde hacerlo: En el Editor VBA → menú Insertar → Módulo. Se abrirá una hoja en blanco. Escribe el siguiente código:
MODULE1.BAS — Encabezado de autor 
' ============================================================
' Módulo: Gestión de Inventario
' Alumno: Tu Nombre Aquí
' Matrícula: Tu Matrícula
' Materia: Computación II
' Fecha: Date()
' Descripción: Formulario VBA para registro de inventario
' ============================================================

Sub AbrirFormulario()
    ' Este procedimiento abre el formulario principal
    ' Se ejecuta cuando el usuario pulsa el botón en la hoja
    frmInventario.Show
End Sub
¿Qué hace Sub AbrirFormulario()?

Sub declara un procedimiento (bloque de código con nombre). AbrirFormulario es el nombre que le damos. frmInventario.Show le dice a VBA que muestre el UserForm que llamaremos frmInventario. Todavía no existe, lo crearemos en la Fase 2.

Programación: Initialize y botón Agregar

Objetivo: Escribir el código VBA que inicializa el formulario (carga las listas) y el que captura los datos del usuario para escribirlos en la hoja de Excel.
1
Abrir el código del UserForm
El código del formulario va dentro de su propio módulo
VBA
○
¿Dónde va el código del formulario?

Cada UserForm tiene su propio módulo de código. Para abrirlo: haz doble clic en cualquier parte del formulario (en el área de diseño, no en un control). Se abrirá el editor con la cabecera Private Sub UserForm_Click(). Borra esa línea y estarás en el módulo correcto del formulario.

Desde los dos menús desplegables en la parte superior del editor puedes seleccionar el control (izquierda) y el evento (derecha) para generar automáticamente la estructura de cada Sub.

📍
Cómo abrir: En el Editor VBA, doble clic sobre el formulario frmInventario en el área de diseño (no en un botón). Se abre el módulo del formulario.
2
Escribir el evento UserForm_Initialize
Este código se ejecuta automáticamente al abrir el formulario
VBA
○
¿Qué es el evento Initialize y por qué es importante?

VBA tiene eventos: bloques de código que se ejecutan automáticamente cuando ocurre algo. UserForm_Initialize se dispara justo antes de que la ventana aparezca en pantalla. Es el lugar ideal para preparar el formulario: llenar los ComboBox con sus opciones, poner la fecha de hoy, limpiar campos residuales, y actualizar el contador de registros. Sin este evento, los ComboBox aparecerían vacíos.

📍
Cómo generar la estructura: En el menú desplegable izquierdo selecciona UserForm, en el derecho selecciona Initialize. Se creará Private Sub UserForm_Initialize(). Escribe el código entre Sub y End Sub:
frmInventario — UserForm_Initialize 
Private Sub UserForm_Initialize()

    ' ── 1. Llenar categorías en el ComboBox
    ' .AddItem agrega cada opción a la lista desplegable
    With cmbCategoria
        .AddItem "Electrónica"
        .AddItem "Herramientas"
        .AddItem "Materia Prima"
        .AddItem "Consumibles"
        .AddItem "Equipos de Seguridad"
    End With

    ' ── 2. Llenar estados posibles del producto
    With cmbEstado
        .AddItem "Disponible"
        .AddItem "Bajo Stock"
        .AddItem "Agotado"
        .AddItem "En Pedido"
    End With

    ' ── 3. Poner la fecha de hoy en el campo txtFecha
    ' Format() da formato legible a la fecha del sistema
    txtFecha.Value = Format(Date, "DD/MM/YYYY")

    ' ── 4. Actualizar el contador de registros en lblTotal
    ' Sheets() accede a la hoja por nombre
    ' Cells(Rows.Count,1).End(xlUp).Row encuentra la última fila con datos
    Dim ws As Worksheet
    Set ws = Sheets("BD_Inventario")
    Dim total As Long
    total = ws.Cells(ws.Rows.Count, 1).End(xlUp).Row - 1
    lblTotal.Caption = "Total: " & total & " registros"

End Sub
Desglose línea por línea

With cmbCategoria … End With: bloque que evita repetir el nombre del control. Cada .AddItem agrega una opción a la lista.

Format(Date, "DD/MM/YYYY"): Date devuelve la fecha actual del sistema. Format() la convierte al formato día/mes/año.

ws.Cells(ws.Rows.Count, 1).End(xlUp).Row: empieza desde la última fila de Excel, sube hasta encontrar la última celda con dato en la columna A. Así sabemos cuántos registros hay. Restamos 1 para no contar el encabezado.

Programar btnAgregar_Click con validación
Valida los campos y escribe el nuevo registro en la hoja
VBA
○
¿Qué hace este código paso a paso?

Paso 1 — Validación: Antes de escribir en la hoja verifica que los campos obligatorios no estén vacíos. Si alguno está vacío, muestra un MsgBox de advertencia y detiene la ejecución con Exit Sub.

Paso 2 — Encontrar la siguiente fila libre: Usa la misma técnica de End(xlUp).Row + 1 para saber en qué fila escribir el nuevo dato.

Paso 3 — Escribir los datos: ws.Cells(fila, columna).Value = ... escribe en la celda específica. La columna 1 es A, 2 es B, etc.

📍
Cómo generar la estructura: En el menú izquierdo selecciona btnAgregar, en el derecho Click. Escribe dentro del Sub:
frmInventario — btnAgregar_Click 
Private Sub btnAgregar_Click()

    ' ── VALIDACIÓN: campos obligatorios no pueden estar vacíos
    ' El operador Or verifica si ALGUNA condición es verdadera
    If txtProducto.Value = "" Or _
       txtCantidad.Value = "" Or _
       txtPrecio.Value = "" Then
        MsgBox "Completa los campos obligatorios (*)", _
               vbExclamation, "Campos requeridos"
        Exit Sub  ' detiene la ejecución aquí
    End If

    ' ── VALIDACIÓN: cantidad y precio deben ser números
    If Not IsNumeric(txtCantidad.Value) Or _
       Not IsNumeric(txtPrecio.Value) Then
        MsgBox "Cantidad y Precio deben ser valores numéricos.", _
               vbExclamation, "Error de formato"
        Exit Sub
    End If

    ' ── REFERENCIA a la hoja de datos
    Dim ws As Worksheet
    Set ws = Sheets("BD_Inventario")

    ' ── ENCONTRAR la primera fila vacía
    ' End(xlUp) sube desde el fondo y encuentra el último dato
    ' +1 da la siguiente fila libre
    Dim UltimaFila As Long
    UltimaFila = ws.Cells(ws.Rows.Count, 1).End(xlUp).Row + 1

    ' ── ESCRIBIR en la hoja (columna por columna)
    ws.Cells(UltimaFila, 1).Value = UltimaFila - 1  ' ID automático
    ws.Cells(UltimaFila, 2).Value = txtProducto.Value
    ws.Cells(UltimaFila, 3).Value = cmbCategoria.Value
    ws.Cells(UltimaFila, 4).Value = CDbl(txtCantidad.Value) ' CDbl convierte a número
    ws.Cells(UltimaFila, 5).Value = CDbl(txtPrecio.Value)
    ws.Cells(UltimaFila, 6).Value = txtProveedor.Value
    ws.Cells(UltimaFila, 7).Value = txtFecha.Value
    ws.Cells(UltimaFila, 8).Value = cmbEstado.Value

    ' ── CONFIRMAR al usuario y limpiar campos
    MsgBox "Registro agregado correctamente.", vbInformation
    Call btnLimpiar_Click  ' reutiliza el código de limpiar (lo haremos en Fase 4)

    ' ── ACTUALIZAR el contador
    lblTotal.Caption = "Total: " & (UltimaFila - 1) & " registros"

End Sub
⚡
Nota: Call btnLimpiar_Click llama al código del botón Limpiar. Ese código lo escribiremos en la Fase 4. Por ahora deja esa línea; no causará error porque el Sub existe (aunque vacío) desde que colocaste el botón.

Programación: Buscar, Limpiar y Cerrar

1
Programar btnLimpiar_Click
Vacía todos los controles del formulario para nueva captura
VBA
○
¿Por qué programar Limpiar antes que Buscar?

El código de btnAgregar ya tiene una línea Call btnLimpiar_Click. Si no programamos Limpiar ahora, cuando Agregar intente llamarlo encontrará un Sub vacío y no limpiará nada. Programamos Limpiar primero para que el flujo completo funcione correctamente.

.Value = "" vacía un TextBox. .ListIndex = -1 deselecciona un ComboBox (ninguna opción seleccionada). .SetFocus pone el cursor en ese campo para que el usuario pueda escribir de inmediato.

frmInventario — btnLimpiar_Click 
Private Sub btnLimpiar_Click()

    ' ── Vaciar todos los TextBox del formulario
    ' .Value = "" asigna cadena vacía (borra el contenido)
    txtProducto.Value  = ""
    txtCantidad.Value  = ""
    txtPrecio.Value    = ""
    txtProveedor.Value = ""

    ' ── Restablecer la fecha al día de hoy
    txtFecha.Value = Format(Date, "DD/MM/YYYY")

    ' ── Deseleccionar los ComboBox
    ' ListIndex = -1 significa "sin selección"
    cmbCategoria.ListIndex = -1
    cmbEstado.ListIndex    = -1

    ' ── Colocar el cursor en el primer campo
    txtProducto.SetFocus

End Sub
2
Programar btnBuscar_Click con bucle For…Next
Recorre cada fila de la hoja buscando el producto escrito
VBA
○
¿Cómo funciona la búsqueda?

El código usa un bucle For…Next que recorre todas las filas de la hoja desde la fila 2 (primera con datos) hasta la última con contenido. En cada iteración usa InStr() para verificar si el texto buscado aparece dentro del nombre del producto (búsqueda parcial: escribir "taladro" encontrará "Taladro eléctrico").

Si encuentra una coincidencia, carga todos los valores de esa fila en los controles del formulario y sale del bucle con Exit For para no seguir buscando innecesariamente.

frmInventario — btnBuscar_Click 
Private Sub btnBuscar_Click()

    ' ── Verificar que el usuario escribió algo en txtProducto
    If txtProducto.Value = "" Then
        MsgBox "Escribe el nombre del producto a buscar.", _
               vbInformation, "Búsqueda"
        Exit Sub
    End If

    Dim ws As Worksheet
    Set ws = Sheets("BD_Inventario")

    Dim Busqueda As String
    Busqueda = txtProducto.Value  ' guardar el texto buscado

    Dim i As Long
    Dim Encontrado As Boolean
    Encontrado = False

    ' ── BUCLE: recorre cada fila desde la 2 hasta la última con datos
    For i = 2 To ws.Cells(ws.Rows.Count, 1).End(xlUp).Row

        ' InStr busca texto dentro de otro texto
        ' vbTextCompare ignora mayúsculas/minúsculas
        ' > 0 significa que encontró algo
        If InStr(1, ws.Cells(i, 2).Value, Busqueda, vbTextCompare) > 0 Then

            ' ── Cargar los datos de esa fila en los controles
            txtProducto.Value  = ws.Cells(i, 2).Value
            cmbCategoria.Value = ws.Cells(i, 3).Value
            txtCantidad.Value  = ws.Cells(i, 4).Value
            txtPrecio.Value    = ws.Cells(i, 5).Value
            txtProveedor.Value = ws.Cells(i, 6).Value
            txtFecha.Value     = ws.Cells(i, 7).Value
            cmbEstado.Value    = ws.Cells(i, 8).Value

            Encontrado = True
            Exit For  ' encontrado: detener el bucle
        End If

    Next i

    ' ── Si no encontró nada, avisar al usuario
    If Not Encontrado Then
        MsgBox "Producto '" & Busqueda & "' no encontrado.", _
               vbInformation, "Sin resultados"
    End If

End Sub

Unload Me es la instrucción para cerrar y destruir el formulario de la memoria. Me es una referencia especial que apunta al formulario actual (es como decir "a mí mismo"). Si usaras Hide en lugar de Unload, la ventana se ocultaría pero seguiría en memoria; con Unload se elimina completamente.
