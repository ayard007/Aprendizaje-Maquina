📐 El Vector de Movimiento

El movimiento en 2D se representa como un Vector2(x, y). Para las 4 direcciones básicas:


↑W / ↑
←A / ←
●Player
→D / →
↓S / ↓

DirecciónTeclaVector2
ArribaW / ↑(0, 1)
AbajoS / ↓(0, -1)
IzquierdaA / ←(-1, 0)
DerechaD / →(1, 0)
ℹ️
Normalización: Cuando se presionan dos teclas simultáneamente (diagonal), el vector resultante tiene magnitud √2 ≈ 1.41. Normalizar lo lleva de vuelta a magnitud 1 para velocidad consistente. Unity tiene Vector2.normalized y .sqrMagnitude > 0 para verificar si hay movimiento.

💻 Crear el Script PlayerController Código
1

Crear el archivo C#

En Project → Assets/Scripts/ → click derecho → Create → C# Script → nómbralo PlayerController. Doble click para abrir en Visual Studio.

2

Variables y referencias (escribe esto primero)

using UnityEngine;

public class PlayerController : MonoBehaviour
{
    // ——— Inspector Variables ———
    [Header("Movimiento")]
    [SerializeField] private float velocidad = 3f;

    // ——— Referencias privadas ———
    private Rigidbody2D rb;
    private Vector2 direccion;
}
💡
[SerializeField] expone la variable en el Inspector sin hacerla public. Así el diseñador ajusta la velocidad sin tocar el código.
3

Método Awake — obtener referencias

    void Awake()
    {
        // GetComponent busca en THIS GameObject
        rb = GetComponent<Rigidbody2D>();
    }
4

Método Update — leer input

Update se ejecuta cada frame (render). Aquí sólo lees input, nunca mueves físicamente.

    void Update()
    {
        // Input.GetAxisRaw evita suavizado (valores exactos -1, 0 o 1)
        float horizontal = Input.GetAxisRaw("Horizontal");
        float vertical   = Input.GetAxisRaw("Vertical");

        direccion = new Vector2(horizontal, vertical).normalized;
    }
5

Método FixedUpdate — aplicar física

FixedUpdate corre a paso fijo (50 Hz por defecto). Aquí aplicas fuerzas y velocidad al Rigidbody.

    void FixedUpdate()
    {
        // Mover directamente con velocity (sin inercia)
        rb.velocity = direccion * velocidad;
    }
⚠️
¿Por qué NO mover en Update? Porque Update depende del framerate. Si el juego corre a 30 FPS vs 120 FPS, el personaje se movería a diferente velocidad. FixedUpdate garantiza consistencia física.
6

🔄 Voltear el sprite según dirección

Para que el personaje "mire" hacia donde se mueve, necesitas voltear el Sprite Renderer horizontalmente:


1

Agregar referencia al SpriteRenderer

    // Añade esta variable junto a las otras
    private SpriteRenderer sr;

    void Awake()
    {
        rb = GetComponent<Rigidbody2D>();
        sr = GetComponent<SpriteRenderer>();
    }
2

Flip horizontal en Update

    void Update()
    {
        float h = Input.GetAxisRaw("Horizontal");
        float v = Input.GetAxisRaw("Vertical");

        direccion = new Vector2(h, v).normalized;

        // Voltear sprite según movimiento horizontal
        if (h != 0)
        {
            sr.flipX = (h < 0); // true = mirando izquierda
        }
    }
3

Detectar dirección predominante (para animaciones)

Agrega una propiedad que indique la dirección principal del último movimiento:

    // Variable para rastrear última dirección
    public Vector2 UltimaDireccion { get; private set; }

    void Update()
    {
        float h = Input.GetAxisRaw("Horizontal");
        float v = Input.GetAxisRaw("Vertical");

        direccion = new Vector2(h, v).normalized;

        if (direccion != Vector2.zero)
        {
            UltimaDireccion = direccion;
        }

        if (h != 0)
            sr.flipX = (h < 0);
    }
📦 Script completo Fase 1
using UnityEngine;

public class PlayerController : MonoBehaviour
{
    [Header("Movimiento")]
    [SerializeField] private float velocidad = 3f;

    private Rigidbody2D  rb;
    private SpriteRenderer sr;
    private Vector2 direccion;

    public Vector2 UltimaDireccion { get; private set; }

    void Awake()
    {
        rb = GetComponent<Rigidbody2D>();
        sr = GetComponent<SpriteRenderer>();
        UltimaDireccion = Vector2.down; // inicia mirando abajo
    }

    void Update()
    {
        float h = Input.GetAxisRaw("Horizontal");
        float v = Input.GetAxisRaw("Vertical");

        direccion = new Vector2(h, v).normalized;

        if (direccion != Vector2.zero)
            UltimaDireccion = direccion;

        if (h != 0)
            sr.flipX = (h < 0);
    }

    void FixedUpdate()
    {
        rb.velocity = direccion * velocidad;
    }
}

🌀 Crear el Blend Tree

Un Blend Tree combina múltiples animaciones basándose en valores float, perfectamente para movimiento omnidireccional.


5

Crear Blend Tree en el Animator

En la ventana Animator → click derecho en el área vacía → Create State → From New Blend Tree. Nombra el estado Walk. Doble click para entrar al Blend Tree.

6

En el Inspector del Blend Tree:

  • Blend Type: 2D Simple Directional
  • Parameters: moveX y moveY
  • Añade motions con el botón +:

7

Transiciones Idle ↔ Walk

Regresa al nivel raíz del Animator. Crea un estado Idle con el clip Idle_Down. Configura transiciones:

  • Any State → Walk: condición isMoving = true
  • Walk → Idle: condición isMoving = false
  • En ambas transiciones: deshabilita Has Exit Time para respuesta inmediata.

⏱ Módulo 7 · 8 Direcciones: código y conexión Animator ~70 min
🔗 Conectar el código con el Animator Código
1

Actualizar PlayerController — agregar Animator

using UnityEngine;

public class PlayerController : MonoBehaviour
{
    [Header("Movimiento")]
    [SerializeField] private float velocidad = 3f;

    private Rigidbody2D   rb;
    private SpriteRenderer sr;
    private Animator       anim;
    private Vector2        direccion;

    // Hashes = más eficiente que strings en cada frame
    private static readonly int HashMoveX     = Animator.StringToHash("moveX");
    private static readonly int HashMoveY     = Animator.StringToHash("moveY");
    private static readonly int HashIsMoving  = Animator.StringToHash("isMoving");

    void Awake()
    {
        rb   = GetComponent<Rigidbody2D>();
        sr   = GetComponent<SpriteRenderer>();
        anim = GetComponent<Animator>();
    }
2

Update — input y actualizar Animator

    void Update()
    {
        float h = Input.GetAxisRaw("Horizontal");
        float v = Input.GetAxisRaw("Vertical");

        direccion = new Vector2(h, v).normalized;

        bool moviéndose = direccion.sqrMagnitude > 0.01f;

        // Enviar parámetros al Animator
        anim.SetBool(HashIsMoving, moviéndose);

        if (moviéndose)
        {
            anim.SetFloat(HashMoveX, direccion.x);
            anim.SetFloat(HashMoveY, direccion.y);

            // Flip horizontal para movimiento lateral
            if (h != 0)
                sr.flipX = (h < 0);
        }
    }

    void FixedUpdate()
    {
        rb.velocity = direccion * velocidad;
    }
}
